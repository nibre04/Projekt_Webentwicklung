Technische Hintergründe und verwendete Ressourcen

1. Projektziel

Der StockTracker wurde als studentisches Projekt im Modul Web-Programmierung entwickelt. Ziel ist es, komplexe Finanzdaten über eine externe API abzurufen und für den Endnutzer in einem modernen, responsiven Interface aufzubereiten.

2. Technologien

HTML5 & CSS3

Struktur und individuelles Modern-Light Styling.

Bootstrap 5

Nutzung des Grid-Systems für volle Responsivität.

JavaScript

Asynchrone API-Abfrage mittels Fetch-API.

3. Verwendete API-Endpunkte

Für die Datenbereitstellung wird die Finnhub API genutzt. Folgende Endpunkte sind implementiert:

GET /stock/profile2?symbol={ticker}

Abruf von Stammdaten: Firmenname, Logo-URL und Branchenzugehörigkeit.

GET /quote?symbol={ticker}

Abruf von Echtzeit-Kursdaten: Aktueller Preis, Hoch-, Tief- und Eröffnungswerte.

4. Hinweis zur Nutzung

Bitte beachten Sie das API-Limit des kostenlosen Plans von Finnhub (max. 60 Anfragen pro Minute). Sollten keine Daten erscheinen, wurde das Limit eventuell kurzzeitig erreicht.
